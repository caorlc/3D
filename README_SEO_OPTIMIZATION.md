## 关键决策点

### 1. 关键词密度：为何0.8-1.2%？
```
低于0.5%：太低，可能无法充分传达关键词信号
0.8-1.2%：最佳实践，既能优化排名又不显得过度
2%以上：警告信号，容易被判断为关键词堆砌
5%+：严重过度优化，Google可能降权
```

### 2. 为何选择这30个关键词？
```
基于：
✓ 竞品研究（Meshy、Tripo、3D AI Studio）
✓ 台湾市场特色用语（列印vs打印、產生vs生成）
✓ 用户搜索意图分析（技术词 vs 商业词 vs 教育词）
✓ 长尾词规律（主词+修饰词组合）

避免了：
✗ 过于宽泛的词（如「模型」单独使用）
✗ 竞争度极高的词（需要专业SEO团队投入）
✗ 不符合台湾用语习惯的词
```

### 3. 为何Hero密度3.5-4.5%最高？
```
原因：
✓ Hero是第一印象，需要强调核心价值主张
✓ 短文本中密度高不会显得过度
✓ 标题和副标题自然位置频繁出现关键词
✓ 用户在Hero部分停留最长，关键词重复不成问题

其他Section为何更低：
✓ Features/FAQ更长，密度高容易被察觉
✓ 需要通过多样化的LSI词来维持相关性
✓ 过高密度会影响可读性和用户体验
```

### 4. 为何台湾用语这么重要？
```
简体直译的危害：
❌ 「生成器」听起来机械 → 「產生器」更自然
❌ 「支持」可能被理解为政治意涵 → 「支援」更准确
❌ 「零基础」显得居高临下 → 「無需經驗」更尊重
❌ 「质量」略显正式 → 「品質」更商业

台湾用语优势：
✓ 提升用户亲近感和信任度
✓ 符合Google本地化算法倾向
✓ 匹配台湾地区搜索习惯
✓ 避免不必要的文化冲突
```

## 常见问题速答

### Q: 如果某个Section关键词密度达不到目标怎么办？
```
A: 不要强行堆砌！改用以下方法：
1. 增加LSI语义词（如用「轉換」替代「產生」）
2. 补充相关应用场景描述
3. 增加技术术语（如STL、OBJ等）
4. 如果确实无法达到，可以整体调整各Section分配

关键：质量 > 数量，自然 > 刻意
```

### Q: 发现过度优化迹象应该怎么办？
```
A: 立即修复，步骤：
1. 定位问题词汇和位置
2. 使用同义词替换其中一些
3. 打散关键词出现的距离
4. 重新计算密度，确保合理范围内
5. 朗读检查，确保可读性恢复

例如：
❌ "3D建模" "3D建模" "3D建模"（连续）
✓ "3D建模" ... "AI生成3D模型" ... "建立3D資產"
```

### Q: 台湾用语不确定时怎么办？
```
A: 参考优先级：
1. 本指南的替换表（第一手参考）
2. Google台湾搜索结果（竞品怎么说）
3. 维基百科繁体中文版（正式定义）
4. 咨询台湾本地人（最后确认）

不要：
✗ 机械直译简体
✗ 随意创造不存在的词
✗ 混用简繁混用
```

### Q: 发布后应该怎样监控效果？
```
A: 分阶段监控：
Day 1：确保页面加载正常，显示无异常
Day 3：检查初步流量变化，对比基线
Day 7：收集用户行为数据（停留时长、跳出率）
Day 30：评估关键词排名变化
Day 60：完整ROI分析，调整策略

重点关注：
- 目标关键词是否有排名进展
- 用户行为指标是否改善
- 转化率是否有提升
- 如无进展，分析原因（算法更新？竞争加剧？）
```

## 注意事项与风险

### ⚠️ 关键风险

1. **过度优化风险**
   - 症状：连续重复关键词、强行插入不相关词、句子扭曲
   - 后果：Google可能降权、用户体验下降
   - 预防：严格遵循密度标准，多读测试

2. **台湾用语错误**
   - 症状：简体直译、混用简繁、文化冲突
   - 后果：用户感知差、本地化失败、信任度降低
   - 预防：使用替换表、咨询本地人

3. **信息不完整**
   - 症状：关键功能或场景未提及、对比优势不明显
   - 后果：用户转化率低、排名效果差
   - 预防：参考英文原文、完整性检查表

4. **格式技术问题**
   - 症状：JSON格式错误、编码问题、链接失效
   - 后果：页面无法渲染、内容显示异常
   - 预防：JSON验证、技术测试、浏览器测试

### ✅ 质量保证

- 每个Section完成后立即统计密度
- 跨职能审批（SEO + 内容 + 技术）
- 至少两轮质量检查
- 发布前最后一轮完整审视
